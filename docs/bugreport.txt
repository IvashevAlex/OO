В тестовой версии 1.0.0.8 при попытке удалить через меню админа юзера с id 5484457194 в терминале
выходит ошибка list index out of range и удаление не происходит, хотя сообщение об удалении происходит
        Ошибка была связана с измененим структуры БД. Запрос на удаление ссылался на несуществующую таблицу

При создании новой БД SSMS на сервере не смог автоматически распознать строку UserMark как битовую (логическую)
        Ситуация разрешилась указанием типа данных как tinyint, а после создания конвертация данных в bit через команду
        ALTER TABLE  WhiteList ALTER COLUMN UserMark bit;
        
При создании рассылки функции расположенные в main запускаются только  один раз. Далее запускается 
цикл бесконечного пуллинга , которы не прирывается.
        Использовать прерывающийся пуллинг, Запуск двух параллельных агентов, Разбиение на потоки
        Решено запуском папллельного процесса в другом файле обрабатывающим рассылку

В рамках версии 1.0.1.7 был переннос SQL команд в отдельны файл. Это привело к ошибкам во взаимодействии с БД
        Принято решение откатить изменения и не вносить данную правку, а продолжить работу с рассылкой
